Git命令出错

        昨天晚上上传博客，到最后一步时候，出现了错误

H:\myblogs\myblog3\public>git push -u origin master
git@github.com: Permission denied (publickey).
fatal: Could not read from remote repository.

Please make sure you have the correct access rights
and the repository exists.

        找了挺多资料，大多数都是更改hosts文件里的github.com的ip地址。但是修改后依然出错。 最后看到一篇关于解决这个错误的文章，解决方法有点不同，提到了用 git remote set-url origin + github仓库地址 的命令。输入上条命令，再运行 git push origin master ，成功把博客上传到github。

  • 推测错误产生原因

    （1）可能需要更新hosts 里的GitHub的ip地址

    （2）太久（20天）没登录GitHub，原本保存的缓存账号信息失效，需要重新更新GitHub仓库链接地址。
